# v0.3.0 — 单线程短期记忆 (ThreadState)

## 版本目标

- 实现当前会话的服务端短期记忆，刷新不丢失对话
- 三层记忆结构：recent messages + summary + pinned decisions
- 模型上下文由服务端 ThreadState 统一构建，不信前端完整历史
- 工具/Agent 只存最终用户可见文本，执行过程不进记忆
- 长对话自动压缩，早期内容归入摘要

## 非目标

- 不做长期记忆或向量记忆
- 不做多会话历史列表（ChatGPT 左侧面板）
- 不做持久化到数据库（内存存储，服务重启清除）
- 不做 Agent 执行轨迹记忆
- 不做 Memory Inspector

## 关键变更

1. 新增 `thread_state.py`：ThreadState + ThreadStore + compact_thread + TextCollectingWriter
2. `app.py` 新增 thread API：POST/GET/DELETE /api/thread
3. `chat_service.py` 集成 ThreadState：上下文构建 + 回合后写入 + 压缩触发
4. `useChat.ts` 增加 thread_id 管理 + 刷新恢复 hydration
5. 三层记忆：recent（≤8条纯文本）+ summary（≤2500字）+ pinned（≤20条）
6. 写入策略：只在回合完成后写一次，流级错误不写入
7. 压缩策略：超 8 条触发模型生成 summary + pinned_decisions
8. 安全降级：memory 失败不影响用户已收到的回答

## 测试计划

- 刷新恢复：对话后刷新页面，消息从服务端恢复
- 长对话压缩：超过 8 条后触发 compaction，早期内容进入 summary
- 工具隔离：工具调用结果不进入 ThreadState，只有最终文本进入
- Agent 隔离：Agent 执行步骤不进入 ThreadState，只有最终回答进入
- 错误降级：流级错误时不写入记忆，不影响下一轮
- 清空对话：创建新线程，旧线程清除

## 交付结果

- 刷新后当前会话能恢复，不是空白聊天
- 长对话后模型看到的是整理过的上下文，不是无限历史
- 工具和智能体的最终回答可以成为聊天记忆，但中间执行态留在运行时边界内
