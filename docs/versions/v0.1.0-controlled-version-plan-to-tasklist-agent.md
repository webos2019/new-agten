# v0.1.0 — 受控版本方案到任务清单 Agent

## 版本目标

- 实现第一个受控 Agent：Version Plan → Tasklist Agent
- 基于 `/tasklist` + `@docs://versions/*.md` 显式入口
- Runtime 读取版本方案 → Agent 生成任务清单草稿
- 确定性工具做结构校验 → 必要时最多自动修正一次
- 最终输出可复制草稿、校验结论和人工确认点

## 非目标

- 不做通用 Agent
- 不做开放式 Planner（让模型自己规划步骤）
- 不做多 Agent
- 不做长期记忆或 RAG
- 不做自动扫描 docs/versions/ 目录
- 不自动发现最新版本方案
- 不自动写入 docs/tasklists/ 目录
- 不从用户目标直接生成 tasklist

## 关键变更

1. Composer 支持 `/tasklist` 斜杠命令
2. 资源菜单支持 `@docs://versions/*.md` 引用
3. Chat Orchestrator 新增 Agent 受控分支（最优先短路）
4. AgentState 管理草稿 v1/v2、校验结果、修正次数（仅本轮内存，不持久化）
5. planExtract 步骤提取版本方案结构化依据
6. validate_tasklist_structure 确定性质量门（检查标题/来源/步骤/勾选项/验证内容）
7. 最多一次自动修正循环
8. 最终输出包含可复制草稿 + 校验结论 + 人工确认点

## 测试计划

- 入口检测：缺少版本方案引用时给出明确提示
- 草稿生成：模型基于 planExtract 结果生成 tasklist
- 结构校验：检查标题、来源 URI、步骤数、勾选项、验证内容
- 修正循环：v1 校验失败 → 修正 v2 → 再次校验（最多一次）
- 最终输出：可复制草稿 + 校验结论 + 人工确认点 + 未自动写入声明

## 交付结果

- Tasklist Agent 完整链路跑通
- Agent 边界可控（入口明确、步骤固定、停止条件清晰）
- 前端可观察 Agent 执行过程（AgentTracePanel + agent-step 事件）
- 最终输出可接力（可复制草稿，留给开发者确认后落地）
